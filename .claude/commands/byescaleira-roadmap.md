# /byescaleira-roadmap

Review, create, or reorganize a `ROADMAP.md` for the current project.

## Steps

1. Read the existing `ROADMAP.md` if it exists.
2. Read `PROPOSAL.md`, `CHANGELOG.md`, and `DECISIONS.md` for context.
3. Build or update the roadmap with exactly these sections:
   - Now (current cycle) — max 3 items, each with a clear done criterion
   - Next (next cycle)
   - Later
   - Maybe
   - Never / deliberately out of scope
4. Ensure every item in Now is achievable within a 6-week cycle.
5. Update the "Last updated" date.
6. Use English and the byescaleira voice.

## Output

A clean `ROADMAP.md` with realistic priorities.
